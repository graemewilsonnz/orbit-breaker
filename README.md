# Orbit Breaker

Orbit Breaker is a desktop-first radial Canvas 2D arcade shooter. The player moves around a fixed
outer orbit, fires inward, survives eight waves, and defeats a three-phase mothership.

Current checkpoint: **M6 — Balance, QA, and release candidate**. The complete eight-wave and
three-phase boss run is protected by a ten-seed release matrix, long-session and rapid-restart
regressions, Chromium and Firefox browser projects, and rolling frame-work, heap, entity, and audio
pressure diagnostics. The M5 presentation, mixer, accessibility, fullscreen, interruption, and
local-high-score work remains intact.

## Requirements

- Node.js 20.19.x or 22.12 or newer
- A current desktop browser

## Development

```sh
npm install
npm run dev
```

Open the local URL printed by Vite. The game uses a deterministic run seed; pass `?seed=1234` to
reproduce a run. During development, `?debug=1` opens the debug panel.

The development panel includes wave select, seed control, first-action timing, enemy-specific damage
attribution, live wave outcomes, wave timing, boss health and phase state, encounter and phase
timers, transition timing, beam and safe-arc counts, and performance counters for repeatable
regression runs.

## Commands

- `npm run dev` starts the development server.
- `npm run build` creates the static production build in `dist/`.
- `npm run preview` serves that production build locally.
- `npm run test` runs unit and deterministic scenario tests.
- `npm run test:watch` runs Vitest in watch mode.
- `npm run test:e2e` runs browser smoke tests.
- `npm run test:e2e:chromium` runs only the Chromium browser gate.
- `npm run test:e2e:firefox` runs only the Firefox browser gate.
- `npm run lint` checks source quality.
- `npm run typecheck` checks strict TypeScript contracts.
- `npm run check` runs typecheck, lint, tests, and the production build.
- `npm run check:release` runs the full engineering gate followed by both browser projects.

## Controls

- Left/A and Right/D: rotate
- Space/Z: fire
- Shift/X: dash
- B/C: panic bomb
- P/Escape: pause or resume
- M: mute or restore audio
- F: enter or exit fullscreen
- S: open settings
- Enter: start or restart

The full implementation plan is in `docs/DEVELOPMENT_PLAN.md`; milestone verification notes are in
`docs/PLAYTESTS.md`.

## Deployment

Run `npm ci`, install the Playwright Chromium and Firefox binaries, and run `npm run check:release`;
then upload the contents of `dist/` to any static host. The host needs no backend, rewrite rule,
environment variable, or runtime package installation. Serve `dist/index.html` over HTTPS with its
generated `assets/` directory unchanged. A double-click standalone bundle is not part of the
approved M6 scope.

To verify the exact production output locally:

```sh
npm run build
npm run preview
```
